## CONCEITOS GERAIS ##

- Tipos de estados (status):
    - Working directory:
        - Modificados (modified - M)
        - Excluídos (deleted - D)
        - Adicionados (added - A)
        - Não mapeado (untracked - U)

    - Staging area:
        - staged (adicionado), prontos para serem versionados (commitados)
        
    - Commited
        - Alterações salvas

- Head: indica a última modificação em uma branch

- .gitignore: arquivo onde são configurados arquivos a serem ignorados pelo versionamento, de acordo com as regras ("locators") listados

- Fork: cópia feita de determinado projeto, com intuito principal de colaboração para com o mesmo.

- Branch: ramificação, com base na versão principal, de modo a isolar o desenvolvimento de determinada funcionalidade específica, não impactando a versão principal durante o processo.

 - Tag: marca uma alteração (pode ser um commit específico) chave para a evolução do projeto, indicando um marco, facilitando encontrar as mudanças significativas durante o desenvolvimento.
